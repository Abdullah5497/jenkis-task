# jenkins-task

01) What is CI/CD ?
CI/CD stands for Continuous Integration and Continuous Delivery/Deployment.
Continuous Integration (CI): Developers frequently merge their code changes into a shared repository. Automated builds and tests run to detect errors early.
Continuous Delivery/Deployment (CD): Ensures that code changes are automatically tested and delivered to production or staging environments quickly and reliably.

02) give me three tools of CI/CD ?

- Jenkins
- GitLab CI/CD
- CircleCI

03) What is Jenkins pipeline?

A Jenkins pipeline is a set of automated processes that defines how your code moves from development to production. It allows you to build, test, and deploy your application automatically in stages.

04) What language is Jenkins based on?

Jenkins is primarily written in Java.

05) What scripting language is Jenkins pipeline syntax based on?

Jenkins pipeline uses Groovy for its scripting syntax.

06) What are the ways you can write pipeline in Jenkins?
- Declarative Pipeline: Simple, structured, and easier for beginners.
- Scripted Pipeline: More flexible and powerful, based fully on Groovy scripting.
